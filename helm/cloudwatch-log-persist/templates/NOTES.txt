== Using External PostgreSQL and S3 ==

To deploy with existing PostgreSQL and S3 services:

  helm install my-release helm/cloudwatch-log-persist/ \
    --set postgres.enabled=false \
    --set postgres.external.host=rds.example.com \
    --set postgres.external.port=5432 \
    --set postgres.external.database=iceberg_db \
    --set postgres.external.username=admin \
    --set postgres.external.password="YOUR_PASSWORD" \
    --set minio.enabled=false \
    --set minio.external.endpoint=https://s3.amazonaws.com \
    --set minio.external.region=us-east-1 \
    --set minio.external.bucketName=warehouse \
    --set minio.external.accessKey="YOUR_KEY" \
    --set minio.external.secretKey="YOUR_SECRET"

Or provide credentials via Kubernetes Secrets:

  # Create external secrets first
  kubectl create secret generic external-postgres-secret --from-literal=password=YOUR_PASSWORD
  kubectl create secret generic external-s3-secret \
    --from-literal=aws-access-key-id=YOUR_KEY \
    --from-literal=aws-secret-access-key=YOUR_SECRET

  # Then deploy with secret references
  helm install my-release helm/cloudwatch-log-persist/ \
    --set postgres.enabled=false \
    --set postgres.external.host=rds.example.com \
    --set postgres.external.existingSecret=external-postgres-secret \
    --set minio.enabled=false \
    --set minio.external.endpoint=https://s3.amazonaws.com \
    --set minio.external.existingSecret=external-s3-secret
